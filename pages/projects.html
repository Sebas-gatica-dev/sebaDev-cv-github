ToDO-React-CRUD-App

This ToDO application is nothing more than an interface for annotating pending tasks, As the tasks are added, they will be displayed as a new row in the table, they can be modified and deleted, the objective of this project is to illustrate in a simple way the basic operation of a CRUD application. Watch an illustrative video

The front of the application is based on React. Bootstrap 5 classes are used for the styles of the components, as well as the dynamic grid functionality that this framework offers us. In addition, Axios will allow us to take what we need to take from the backend, in order to map it in the table.

The back-end is worked with the Express framework, using the cors middleware, through sequelize, a connection to a relational database is established, the table model and the corresponding routes are instantiated, and the different CRUD functions are created, through which the table is accessed. respective table.

Crypto-React-Axios

In this project, through axios we call the coingecko API, a platform for exchanging and carrying out operations with cryptocurrencies, and they also offer free services to access cryptocurrency quotes. Watch an illustrative video

The front of the application is based on React. Bootstrap 5 classes are used for the styles of the components, as well as the dynamic grid functionality that this framework offers us..

It does not have a backend as such, the data is hosted on the server side , the api is accessed with axios, the endpoint that is called is 'https://api.coingecko.com/api/v3/coins' , from the JSON that the API returns, the data is mapped, in the Bootstrap table.

Image search engine in react

This application is an image search engine that consumes the API of the unsplash page, a page similar to Pinterest , A search filter is generated, through which results will be displayed in relation to those written in the search input. Watch an illustrative video

The front of the application is based on React. The styles were worked with pure css.

the application does not have a backend as such, the information is processed from the server side, using Formik, the information received by the input is handled, through Fetch it connects to the api and the content received by the input is applied to the end of the enpoint, so that the api performs the search.
